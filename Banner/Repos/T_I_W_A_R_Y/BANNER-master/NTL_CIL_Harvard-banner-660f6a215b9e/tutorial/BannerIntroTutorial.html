About the BANNER Tutorial: 

This BANNER tutorial is intended to help beginners in the Natural Language Processing (NLP) domain to understand and work with the Biomedical Named Entity Recognition tool.

This tutorial explains:
  1. Named Entity Recognition 
  2. The BANNER tool 
  3. Steps to install and run BANNER on Windows (Eclipse IDE & Window Command prompt) 
  4. Steps to install and run BANNER on the Linux platform (shell terminal) 
  5. Training and testing corpora and evaluation (calculation of precision, recall and F-measure) using SVN and CVS versions.

Named Entity Recognition concept:

What is Named Entity Recognition?

Named Entity Recognition (NER) is the process of entity detection and extraction in text. Here entities can refer to organization, location, geographic areas, persons, occupations, devices, players, quantities, monetary values, percentages, etc.

For example, the input free text to NER is:
"Mr. Mathew works as a manager in an Indian company, Cognizant Ltd. ",
Entities that can be detected and extracted by NER processing are as below:
Mr. Mathew - person
manager - occupation
Indian - country
Cognizant Ltd. - organization

In the Biomedical domain, NER is very useful to process the biomedical text and extract the entities like gene names, drugs, deceases, disorders, molecules, etc.

Example: "many cytochrome P450 enzymes are transcriptionally suppressed by inflammatory stimuli, down-regulation of CYP2B protein by the inflammatory cytokine interleukin (IL)-1ß"
Tagged entities:
cytochrome P450 enzymes - gene
CYP2B protein - gene
cytokine interleukin (IL)-1ß - gene

In the same above example if gene names are to be tagged/annotated manually, an expert who can identify all gene names is needed. Hence there is a need for a tool which can automatically tag the entities in the text. Name Entity Recognition (NER) does the that tagging function.

What is BANNER? 

BANNER is Biomedical Named Entity Recognition tool implemented in Java. It is freely-available, open source system [1].

Input to BANNER - free text that we need to tag depending upon configuration file.

Output of BANNER - tagged entities.

Why do we need BANNER when similar tools such as ABNER and LingPipe already exist? 

Currently existing widely used and open source systems for gene NER include ABNER and LingPipe. Improved and enhanced methods of gene NER beyond those systems has largely remained in literature without being available as code that can be downloaded and run. Due to the advantages those methods, source code that applies them is quite valuable for Natural Language Processing researchers. Such code can serve as a new benchmark for how such NER can perform. The availability of that code enables software re-use and can help researchers to focus on problems beyond recreating such code. Banner provides a updated benchmark for NER performance.

Banner's precision, recall and therefore F-measure (81.96%) compares well with that of ABNER (F-Measure 78.30%) and LingPipe (F-Measure 64.95). [2]

How to install BANNER from SVN in the Windows platform?

(Download page : http://ant.apache.org/bindownload.cgi)

Building and Installing Banner In Linux

Prerequisite Software List

Java 1.6 Development Kit (JDK)
Ant
Subversion
Step 1 - Download the source code

The latest source code for Banner is available in a Subversion repository hosted by Sourceforge. Start by opening a Linux terminal window, navigating to the parent directory of where you want the Banner code to be downloaded, and run the following command:

svn co https://banner.svn.sourceforge.net/svnroot/banner banner

The command should take about a minute to execute on a broadband connection.

Step 2- Build the source code

Navigate to the checkout directory, and the subdirectory ‘trunk’. Then, execute the following command to use the ‘ant’ build system to build banner:

ant -buildfile build_ext.xml

If the compilation is successful, the “banner.jar” library file and all of its dependencies will be present in the ‘lib’ subdirectory. Type the following to test the program:

java -cp 'lib/*' banner.eval.BANNER

If there are no problems with missing classes or dependencies, you should receive output similar to this one:

Usage: banner.sh <command> <configuration> <parameters>
Exceute "banner.sh help" for details


Note that Banner needs to be executed with the build directory being the current directory, in order to find the metadata that it expects to run properly. Failing to do so will cause exceptions with all functions.



Using the SVN version of Banner as of 11/25/11 in Ubuntu Linux 11.10:

Training:

To train a model with Banner, the class located at /banner/eval/BANNER.java should be run as described above in "Step 2-Build the source code".  Some example command line arguments that can be used for training are “train config.xml” or “train config.xml 0.2”.

In the arguments, config.xml should be the name of the configuration file that Banner will use.  Some existing configuration files are in the directory “config”.  One of those existing configurations can be copied into a new file to be used as a template for creating a new configuration file.

Certain data files should be supplied for use as the training data.  Example datafiles that can be used to train a model are the BioCreative 2 Gene Mention dataset.  Training a new model with Banner is described later in this document.  Using Banner to train the BioCreative 2 Gene Mention dataset shows ways to train a model in Banner that can also be applied to a new corpus.  The BioCreative dataset can be downloaded from http://sourceforge.net/projects/biocreative/files/biocreative2entitytagging/1.1/ .

Another corpus that can be trained with Banner is the Arizona Disease Corpus (AZDC).  To train with that corpus, arguments used for the program can be “train config/banner_AZDC.xml”.  In the banner_AZDC.xml file settings should be specified for the file location of the AZDC.

In the config.xml file, the configurations for <sentenceFilename>, <mentionTestFilename>, and <mentionAlternateFilename> should be set to the appropriate locations on user’s machines.  The sentencesFilename is the name of the input file with sentence IDs and sentences to apply NER on.  mentionTestFilename and mentionAlternateFilename are files that contain annotated mentions for the sentencesFilename.  More information about those file path settings can be found in README.GM which is in the Biocreative data set.  Also, the creating a corpus for Banner section later in this document further describes those files.  Additional configuration setting can be set according to user’s preferences in the config.xml file.

Another filepath setting that should be set correctly is the thesaurus for distributional semantics features in /banner/tagging/pipe/SimFind.java.  By default it is set to “dict/sample_Thesarus_proteins_2000d_5.txt”.  However, it may need to be changed to the name of the thesarus in the “dict” directory.  That name may be “sample_Thesarus_proteins_2000d_20.txt”.

An optional third argument can be used with the training of a model by Banner.  That argument represents a double number type that specifies the amount of the corpus to use for training.  For example, applying the arguments “train config.xml 0.2” will use 0.2 as a percentage of the corpus to use for training.  The third argument should be a double value.  If no third argument is specified, all of the corpus is used for training.

Creating a corpus for training a model with Banner:
A new corpus used with Banner must be in the proper format for Banner to train a NER model from it.  Users must find ways external to Banner to convert their corpus into one compatible for Banner because Banner does not currently supply a conversion tool for that.  The corpus’s “sentenceFilename” file should contain the format of “SentenceID<Tab>Sentence” .  The “mentionTestFilename” and “mentionAlternateFilename” files should contain annotated mentions in the format “SentenceID|StartOffset1 EndOffset1|OptionalText”.

Once the new corpus is in the correct format, the corpus’ files can be entered into Banner along with the same configuration used to train the Biocreative data set.  Using the new files with the same configuration allows for a new NER model to be trained with Banner.

Testing:

The class “/banner/eval/BANNER.java” should be run to test Banner’s NER on a model.  Finding test evaluation results for a model can be done with sample parameters such as “test config/MyUserConfig.xml 0.01”.  The evaluation report that will be generated will include statistics such as F-Score for NER performance as well as others.  The evaluation scores will be reported in standard output of the program.

In the arguments, MyUserConfig.xml is a configuration file which can be created in ways explained in the Training portion of this tutorial.  The model used for testing can be set in the modelFilename parameter in that file.  The number at the end of the argument is optional and specifies the percentage of the corpus to be used for testing.

Using the CVS version of Banner as of 11/25/11 in Ubuntu Linux 11.10:

Training:

Banner can be trained in the Biocreative 2 gene mention corpus format by running the class located at /src/bc2/TrainModel.java .  The arguments that should be used with running the class are “banner.properties sentences.txt tags.txt model.bin” .  Each argument should be customized to user’s files.  Also a number in double format (e.g. 0.25) can be specified as an additional argument that determines the amount of the corpus to be used for training.

The banner.properties file should be customized according to the preferences of users and their local computer configurations.  Descriptions are included in the properties file of variables that can be customized inside of the file.

In the arguments used to train a model, the sentences.txt file should be the unlabeled sentences with their sentence IDs.  The tags.txt file should be the annotations for those sentences.  The formatting of those files is described further in the “Creating a corpus” section of this tutuorial.  The “model.bin” file is the location of the file that will be the model that is created.

Testing:

Testing of a model in the same format as the Biocreative II Gene Mention corpus can be performed by running the src/bc2/TrainModel.java class.  Example arguments that can be used with that class are
“banner.properties sentenceFilename mentionTestFilename mentionAlternateFilename nerModel.bin outputFolder”.  The sentence, mention, and model files are the same file types as described in the training section in the instructions for the SVN version of the program.  The output folder is a location where files such as results from the testing are written.

Results such as precision, recall, and F-score can be seen in the standard output from the program.  The program will also write to standard output the progress it makes with running the tests while it is testing.  That standard output can be checked before the program is done returning results to see if all tests appear to be running with no errors from missing files or other things.
